## US10 — Politique de sécurité mobile

### Objet

Ce document définit les règles applicables au développement, à la distribution et à l'usage des applications mobiles dans le périmètre de l'organisation.

### Développement

Toute application mobile doit être développée en conformité avec l'OWASP Mobile Security Testing Guide (MSTG). En particulier :

- Aucune clé API, secret, ou URL interne ne doit être compilée en dur dans le binaire.
- Le mode debug doit être désactivé dans les builds de production. Les activités de développement (`DevSettingsActivity`, menus debug) doivent être absentes ou désactivées.
- Les communications réseau doivent être intégralement chiffrées (TLS 1.2 minimum, recommandation TLS 1.3). Le cleartext est interdit sauf cas documenté et approuvé.
- Le certificate pinning est obligatoire pour les endpoints bancaires et d'authentification.
- Les données sensibles ne doivent pas être écrites en clair sur le stockage externe.

### Distribution

- Les applications sont distribuées exclusivement via les stores officiels (Google Play, Apple App Store) ou via un MDM interne.
- Chaque release doit être signée avec une clé RSA 2048 bits minimum, schéma v2 ou v3 obligatoire.
- Un processus de revue sécurité (SAST + analyse manuelle) est requis avant toute publication.

### SDKs tiers

- Tout SDK tiers intégré dans une application traitant des données financières ou personnelles doit faire l'objet d'une analyse de risque documentée.
- Les SDKs dont l'origine ou le traitement des données ne peut être garanti conforme RGPD sont interdits.
- La liste des SDKs approuvés est tenue à jour par l'équipe sécurité.

### Permissions

- Chaque permission Android déclarée doit être justifiée par un besoin fonctionnel documenté.
- Les permissions dangereuses (`READ_CONTACTS`, `CAMERA`, `LOCATION`, etc.) font l'objet d'une revue spécifique.
- Le principe de moindre privilège s'applique.

### Terminaux

- Les terminaux utilisés pour le développement et les tests ne doivent pas être rootés ou jailbreakés.
- L'installation d'APK hors store (`sideloading`) est réservée aux environnements de test isolés.
- Les terminaux professionnels accédant à des applications sensibles doivent être enrôlés dans le MDM.
