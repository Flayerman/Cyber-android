## US11 — Choix d'une solution MDM

### Présentation des solutions comparées

Deux solutions MDM ont été évaluées : **Microsoft Intune** et **VMware Workspace ONE** (anciennement AirWatch).

### Tableau comparatif

| Critère | Microsoft Intune | VMware Workspace ONE |
|---|---|---|
| Modèle de déploiement | Cloud (SaaS Azure) | Cloud ou on-premise |
| Intégration écosystème | Microsoft 365, Entra ID, Defender | Multi-éditeur, VMware |
| Support Android | Android Enterprise complet | Android Enterprise complet |
| Support iOS | Oui | Oui |
| Gestion des applications | Oui (MAM/MDM) | Oui (MAM/MDM/MCM) |
| Conditional Access | Natif (Entra ID) | Via intégration IdP |
| Conformité RGPD | Données hébergées en EU (option) | Données hébergées en EU (option) |
| Coût | Inclus dans M365 E3/E5 | Licence séparée |
| Complexité de déploiement | Faible à modérée | Modérée à élevée |
| Reporting sécurité | Intégré (Defender for Endpoint) | Intégré (Workspace ONE Intelligence) |

### Recommandation

**Microsoft Intune** est recommandé pour les organisations déjà sur Microsoft 365. L'intégration native avec Entra ID (anciennement Azure AD) simplifie la gestion des identités et le Conditional Access. Le coût est nul si la licence M365 E3 est déjà en place.

**Workspace ONE** est pertinent pour les environnements hétérogènes (multi-OS, on-premise critique), ou pour les organisations ayant déjà une infrastructure VMware existante.

Dans le contexte de cet audit — une application bancaire Android — Intune offre le meilleur rapport fonctionnalité/coût/complexité, notamment pour appliquer des politiques de conformité (PIN obligatoire, chiffrement, interdiction de sideloading) et bloquer l'accès aux ressources internes depuis des terminaux non conformes.

### Politiques MDM recommandées pour l'application Nickel

```
- Chiffrement du stockage obligatoire
- PIN / biométrie obligatoire
- Version Android minimum : 10 (API 29)
- Sideloading interdit
- Applications provenant de sources inconnues : bloquées
- Accès conditionnel : terminal non-conforme → accès révoqué
- Détection de root : terminal rooté → accès bloqué
```
